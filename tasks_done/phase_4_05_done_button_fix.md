### Истинный механизм (доказано read-only дискавери)

День 1208 уходит из «Запланированные» через **завершение CRM-дела**:
- `PROVIDER_ID="CRM_TODO"` (из `config.activity_provider_id`)
- `OWNER_TYPE_ID=1208`, `OWNER_ID=<id дня>`
- Завершение: `crm.activity.update id=<id> fields:{COMPLETED:"Y"}`

Создание учёта 1218 и закрытие CRM-дела — **два независимых действия**.

Доказательная база:
- День id=271557 (26.06): учёт [163736] уже есть, дело id=169987 `COMPLETED="N"` → «Запланированные».
- День id=267932 (22.06, закрытый): дело `COMPLETED="Y"`.
- День id=254912 (01.06, вне окна): дел нет.
