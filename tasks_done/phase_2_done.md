### 2_01 — Чтение дней «Рабочий день» (1208) за период

Реализован модуль `src/workday.py`:
- Датакласс `WorkdayDay`: поля `id`, `date`, `title`, `employee_id`, `works_ids`, `raw`.
- Функция `read_days(b24, config, date_from, date_to)`: вызывает `crm.item.list` с фильтром по `ufCrm46_1742342657` (`>=date_from`, `<=date_to`), сортировкой `id desc`, полной пагинацией (`item_list_all`). Проверено на 247 записях — пагинация не теряет данных.
- `works_ids` заполняется из `ufCrm46_1742997115`; дата дня — из `ufCrm46_1742342657` через `_coerce_date`, запасной источник — `extract_date(title)`.

### 2_02 — Чтение связанных учётов (1218)

- Датакласс `WorkLog`: поля `id`, `title`, `parent_day_id`, `contract`, `description`, `hours`, `result`, `raw`.
- Функция `read_logs(b24, config, day_ids)`: собирает все ID учётов из `works_ids` дней, запрашивает `crm.item.list` с `filter: {"@id": [...]}` чанками по 50 ID, собирает `WorkLog` для каждой записи.
- Каждый `WorkdayDay` получает атрибут `logs: list[WorkLog]` через `link_logs(days, logs)`.

**Закрытый открытый вопрос — «Информация о проделанной работе»** (задокументирован в PRD §11):

На реальной записи `entityTypeId=1218` (id=156084):
- `ufCrm48_1744239302` («Описание задачи») = краткий код-детализация, например `"AI.12.08, DO.20.16"`.
- `title` учёта = человекочитаемое наименование задачи, например `"Общие задачи подразделения"`.
- `ufCrm48_1743029170` («Итог») = числовое значение (равно часам), а не текст работы.

Решение заказчика: в выгрузку включить **обе** колонки:
1. «Описание задачи» = `title` учёта (имя задачи, читаемое).
2. «Информация о проделанной работе» = `ufCrm48_1744239302` (код-детализация).

### 2_03 — Excel: основной лист

Модуль `src/export_excel.py`, функция `write_main_sheet(ws, days)`:
- Лист «Рабочий день».
- Колонки (6, по правке заказчика от 2026-06-25): Дата | Название дня | Сотрудник | Описание задачи | Информация о проделанной работе | Часы. Колонка «Договор» убрана по запросу заказчика; поле `log_contract` (`ufCrm48_1742996936`) оставлено в `config.yaml`/модели `WorkLog`, но в выгрузку не выводится.
- Колонка «Сотрудник» резолвится в ФИО через `resolve_users` (см. ниже).
- День без учётов — одна строка с пустыми полями учёта.
- День с несколькими учётами — одна строка на учёт (подтверждено на реальном примере, см. ниже).
- Жирный заголовок; формат даты `дд.мм.гггг`; авто-ширина колонок.

**Подтверждение многострочности учётов (правка заказчика от 2026-06-25).** Реальный read-only прогон `export 2025-12-15..2025-12-20`: день `1208` id=149422 («Ильдар Сабиров | 18.12.2025») имеет два учёта `1218` (id=81277, id=81276), оба «Общие задачи подразделения | 4:00». В выгрузке день корректно развёрнут в **2 строки** (по 4 часа), значения «Информация о проделанной работе» у строк различаются («Протоколы ПИ ПЛК+ИВК» и «Стенд AIAO — РМ ТС-001.01 …»). Механизм `works_ids → read_logs → link_logs` уже обрабатывал множественные учёты — правок не потребовалось.

### 2_04 — Excel: лист группировки

Функция `write_grouping_sheet(ws, days)`:
- Лист «Группировка».
- Ключ группы — `WorkLog.description` (`ufCrm48_1744239302`); пустое значение → `"(без описания)"`.
- Агрегаты на строку: кол-во записей, сумма часов, список дат.
- Контрольная проверка (факт): сумма часов по группировочному листу = сумме на основном листе (135.0 == 135.0).
- Текущая сортировка — по описанию (алфавитная); порядок сортировки открыт для уточнения заказчиком.

## Уточнение метамодели (закрытый открытый вопрос 2_02)

Поле «Информация о проделанной работе» в UI не является отдельным полем `1218`. Факт по записи id=156084:

- `ufCrm48_1744239302` («Описание задачи») — код-детализация (краткий технический код, например `"AI.12.08, DO.20.16"`), НЕ человекочитаемое имя.
- `title` учёта — полное имя задачи/договора, например `"Общие задачи подразделения"`. Это и есть читаемое «Описание задачи» с точки зрения пользователя.
- `ufCrm48_1743029170` («Итог») — числовое поле (дублирует часы), не содержит текст работы.

Вывод занесён в PRD §11 («Уточнения метамодели по факту дискавери»).

## Решения заказчика (зафиксированы)

1. **Колонка «Информация о проделанной работе»** — в выгрузке оставлены обе колонки:
   - «Описание задачи» = `title` учёта 1218 (человекочитаемое имя задачи).
   - «Информация о проделанной работе» = `ufCrm48_1744239302` (код-детализация из поля «Описание задачи» в CRM).
2. **Колонка «Сотрудник»** — резолвится в ФИО через read-only `user.get` с кэшем. В выгрузке отображается «Сабиров Ильдар Касыймович» вместо числового ID 1244. Один HTTP-вызов `user.get` на весь период (кэш по user_id).
