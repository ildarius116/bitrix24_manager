## Что сделано

Реализована функция отбора дней «Рабочий день» (1208), которым нужно создать учёт:
в пределах 4-дневного окна и с пустым столбцом «Работы/задачи за день».

### Реализация

В `src/workday.py` добавлена функция:

```python
def select_candidates(
    days: List[WorkdayDay],
    cfg: Config,
    today: date,
    *,
    limit: int = 5,
) -> List[WorkdayDay]
```

- В импорт `from datetime import date` добавлен `timedelta`.
- Логика фильтров по порядку:
  1. **Топ-`limit` (FR-2.1.1):** рассматриваются первые `limit` дней (дефолт 5). Дни приходят
     отсортированными `id desc` из `read_days` (`order={"id":"desc"}`).
  2. **Нет даты (FR-2.1.2):** `day.date is None` → пропуск «нет даты». Дата дня уже распарсена
     в `WorkdayDay.from_item` из `ufCrm46_1742342657`, запасной источник — разбор из title.
  3. **4-дневное окно (FR-2.1.3):** оставить дни, у которых
     `today - cfg.edit_window_days <= day.date <= today` (включительно с обеих границ; будущие
     даты `> today` тоже отсекаются). Семантика совпадает с `within_edit_window` из `src/dates.py`.
  4. **Пустота (FR-2.1.4/5):** оставить только дни с пустым `works_ids` (непустой = «уже заполнено»).
- Каждый пропуск и каждый отобранный кандидат логируются через `log.info` (логгер «workday») с
  понятной русской причиной; в конце — итоговая строка отбора.
- Функция полностью **read-only / без сети**: работает с уже прочитанными днями, REST-вызовов
  внутри нет. Коды полей не хардкодятся — используется только `cfg.edit_window_days`.

### Тесты

В `tests/test_workday.py` добавлено **13 юнит-тестов** на синтетике (без сети). Покрыты случаи:
- happy path (корректный день в окне с пустым works_ids возвращается);
- граница окна `today−4` проходит, `today−5` — нет;
- будущая дата (`> today`) отсекается;
- `date=None` отсекается;
- непустой `works_ids` отсекается, пустой — проходит;
- дефолтный `limit=5` режет 7 подходящих дней до 5; явный `limit=2`;
- 6-й день в окне недостижим из-за исчерпанного `limit`;
- комбо смешанных причин (будущая + заполнено + None + старее окна + один кандидат);
- пустой вход → пустой результат;
- все 5 дней в окне с пустым works_ids проходят.

Добавлен хелпер `_bare_day(d, *, day_id, works_ids)` рядом с `_day` (позволяет независимо
контролировать `works_ids` без реальных `WorkLog`). `Config` заменён лёгким
`types.SimpleNamespace(edit_window_days=4)` (функция использует только это поле). `today`
зафиксирован как `date(2026, 6, 25)` для детерминизма.
