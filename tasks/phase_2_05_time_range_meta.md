# phase_2_05 — Временные рамки данных + лист метаданных

**Статус:** TODO

## Цель
Вычислить фактические временные рамки выгруженных данных и зафиксировать их.

## Объём работ
- Функция `data_time_range(days)` → (min_date, max_date, число дней с датой).
- Лист «Метаданные»: всего строк, строк с датой, дата min/max, заданный период, признак расхождения.
- Лог + предупреждение, если фактические рамки **уже** заданного периода (возможна обрезка пагинацией / отсутствие данных).

## Критерии приёмки (DoD)
- На тестовых данных рамки совпадают с реальными min/max.
- При периоде шире данных выводится предупреждение.
