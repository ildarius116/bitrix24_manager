## Цель
Получить значения для нового учёта: дефолт из `config.yaml` + интерактивное подтверждение.

## Объём работ
- Дефолты: `defaults.task_description` = «Общие задачи подразделения», `defaults.hours` = 8.
- Интерактив (если `runtime.interaction`): по каждой записи показать предлагаемые значения; ввод — Enter (принять), новое значение (переопределить), `skip` (пропустить), `abort` (прервать).
- Опция «применить ко всем» (запомнить введённое для остальных записей пакета) — согласовать.
- Режим `--no-interaction`: дефолты без вопросов.
- Валидация часов (число > 0).

## Критерии приёмки (DoD)
- Возвращает `(описание, часы)` либо сигнал skip/abort.
- `--no-interaction` использует дефолты; интерактив корректно обрабатывает все варианты ввода.

## Открытые вопросы
