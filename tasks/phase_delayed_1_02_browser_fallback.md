# phase_delayed_1_02 — Браузерный fallback (отложено)

**Статус:** DELAYED
**Связано:** PRD §3 (REST-only решение)

## Идея
Если выяснится (phase_4_01), что кнопка «Выполнено» запускает критичный БП без REST-аналога,
и `crm.item.add` недостаточно для «заполнения» дня — реализовать узкий браузерный fallback
(Playwright) только для этого шага.

## Когда брать
Только при подтверждённой невозможности закрыть «Выполнено» через REST/bizproc.
По умолчанию НЕ делаем (2FA, хрупкость). Требует отдельного решения заказчика.
