## Цель
Реализовать интерактивное подтверждение записи по дню (FR-2.1.10) через модальный диалог и
наполнить меню (открытие выгрузки, выбор темы, масштаб, «О программе»).

## Объём работ
- `gui/confirm_dialog.py` — `QDialog`:
  - заголовок «Подтверждение записи»; строка «День: дд.мм.гггг (id N)»;
  - «Описание задачи» (`QLineEdit`, предзаполнено дефолтом/значением дня);
  - «Количество часов» (`QDoubleSpinBox`, предзаполнено дефолтом);
  - кнопки [Пропустить] [Применить ко всем] [Отмена] [OK];
  - возвращает структуру выбора `{action, description, hours}` где action ∈
    {ok, apply_all, skip, abort}.
  - Диалог вызывается из main thread по сигналу `confirm_requested` воркера; результат
    передаётся обратно и разблокирует воркер (`QSemaphore`/`QWaitCondition` из gui_03).
- Меню (`main_window.py`):
  - Файл → «Открыть выгрузку (Excel)» (QFileDialog по `out`), «Выход»;
  - Вид → «Тема» (Светлая/Тёмная/Системная → `theme.apply`), «Масштаб»;
  - Справка → «О программе» (QMessageBox: название, версия, краткое описание).

## Критерии приёмки (DoD)
- При Dry-run ВЫКЛ и Авто-режим ВЫКЛ по каждому дню всплывает диалог; ветки OK/Пропустить/
  Применить-ко-всем/Отмена корректно влияют на `run_fill` (filled/skipped/aborted; «ко всем»
  больше не спрашивает).
- Переключение темы из меню перекрашивает UI; лог-панель остаётся тёмной.
- «Открыть выгрузку» открывает файл; «О программе» показывает диалог.

## Открытые вопросы
- Маппинг кнопки «Применить ко всем» с отредактированными значениями обрабатывается в воркере
  (кэш), а не токеном `all` collect_values — согласовано в плане.

## Артефакты
`gui/confirm_dialog.py`, правки `gui/main_window.py` (меню, проводка диалога к воркеру).
