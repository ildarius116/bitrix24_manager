# Фаза 8_01 — «Индивидуально»: выбор дней реально управляет записью + блокировка недоступных

## Проблема

В методе «Индивидуально» (GUI, режим «Редактирование») чекбоксы дней и инлайн-поля
описания/часов **декоративны**: `ControlPanel.selected_days()` / `per_day_values()`
определены, но не вызываются нигде. Реальную запись ведёт `run_fill()`, который сам
отбирает топ-5 (`select_candidates`) и в интерактиве показывает `ConfirmDialog` по каждому
дню, игнорируя левую панель.

## Требуемое поведение (только метод «Индивидуально»)

1. **Пишутся ТОЛЬКО отмеченные дни**, каждый — значениями из СВОИХ инлайн-полей.
2. **Per-day `ConfirmDialog` убирается** — источник выбора/значений — левая панель.
3. Недоступные дни среди найденных ~5 (уже заполнены / не рабочий тип / вне окна 4 дней /
   нет записи «Рабочий день») — **disabled**, галку менять нельзя.

«По-умолчанию» НЕ меняется. Общий диалог-предупреждение боевой записи
(`_confirm_live_write`) остаётся для обоих методов.

## Жёсткие ограничения

- **`src/` НЕ трогать.** Всё делается в `gui/`.
- Секрет вебхука не логировать/не показывать (CLAUDE.md §4).
- Потокобезопасность Qt: воркеры не трогают виджеты, только сигналы; обновления UI —
  в main thread.

## Часть B — выбор/значения → запись (без правки `src/`)

Механизм — существующий перехват `builtins.input` в `FillWorker` (`interaction=True`
заставляет `run_fill`→`collect_values` звать `input()` по каждому дню).

- `MainWindow._start_fill` при методе «Индивидуально» передаёт в `FillWorker`
  `selected_days` (set `date`) и `per_day_values` (`dict[date, (str, float)]`) из `ControlPanel`.
- `FillWorker.__init__` принимает и хранит эти структуры (по умолчанию — None/пусто, чтобы
  не ломать «По-умолчанию» и export).
- `FillWorker._fake_input` (СИНХРОННО, без `confirm_requested`/поп-апа) для «Индивидуально»:
  - промпт содержит «Описание» → определить дату дня по `self._last_day_date`
    (его уже наполняет `_StdoutTap` в формате `dd.mm.yyyy`):
      - дата НЕ в `selected_days` → вернуть `"skip"`;
      - дата в `selected_days` → выставить `self._current_hours` из инлайн-часов дня и
        вернуть инлайн-описание дня (из `per_day_values`);
  - промпт «Количество часов» → вернуть `self._current_hours`;
  - промпт «ко всем» → `"N"`.
- Сопоставление дат: ключи `per_day_values`/`selected_days` — `date`; `_last_day_date` —
  строка `dd.mm.yyyy`. Парсить строку в `date` для сравнения (устойчиво к формату).
- Убрать использование per-day диалога из интерактива: `_start_fill` больше не подключает
  `confirm_requested`→`_on_confirm_requested`; блокирующее ожидание (`QWaitCondition`) в этом
  пути не нужно. `provide_confirmation` может остаться как no-op (его дёргает `closeEvent`).
  Модуль `gui/confirm_dialog.py` — оставить или удалить (согласовать с reviewer); мёртвую
  проводку в `main_window.py` убрать.
- `_reread_guard` (`src/fill.py`) остаётся страховкой — не дублировать.

## Часть A — блокировка недоступных чекбоксов

Источник истины «fillable» = `src/workday.py::select_candidates`. Классификация — на клиенте
по прочитанным `WorkdayDay`, без дублирования REST-логики записи.

- `gui/worker.py`:
  - Чистая функция `classify_day_states(days, cfg, today) -> dict[str, dict]`
    (ключ — ISO-дата `date.isoformat()`; значение `{"fillable": bool, "reason": str}`).
    Причины: `нет записи «Рабочий день»` (даты нет среди `days`), `не рабочий день`
    (не проходит `_day_type_allowed`/`cfg.day_type_work_ids`), `вне окна 4 дней`,
    `уже заполнен (N)` (works_ids непуст), иначе `fillable=True, reason=""`.
    Окно дат — `today−cfg.edit_window_days … today` (те же 5 строк, что рисует панель).
  - `DayStatesWorker(QThread)`: `B24(config)` → `_smoke` → `read_days(b24, cfg,
    today−edit_window_days … today, day_type_ids=cfg.day_type_work_ids)` →
    `classify_day_states`. Сигналы `states_ready(object)` (dict) / `states_failed()`.
    Read-only: в прод не пишет, виджеты не трогает.
- `gui/control_panel.py`:
  - Сигнал `day_states_requested = Signal()`.
  - Кнопка **«↻ Обновить»** в шапке секции «ДНИ ДЛЯ ЗАПОЛНЕНИЯ» — видна ТОЛЬКО в
    «Индивидуально».
  - `set_day_states_loading()` — чекбоксы disabled + подпись «загрузка…».
  - `apply_day_states(mapping: dict)` — по дате строки: `fillable` → enabled; иначе →
    disabled + `setChecked(False)` + подпись/tooltip с причиной. Применять ТОЛЬКО в
    «Индивидуально» (в «По-умолчанию» — не блокировать).
  - `apply_day_states_error()` — фолбэк: вернуть чекбоксы в активное состояние + подпись
    «не удалось получить статусы».
  - Эмитить `day_states_requested`: при смене метода на «Индивидуально» (режим fill) и по
    кнопке «Обновить». При переходе в «По-умолчанию» — снять блокировку (все enabled).
  - Сохранить существующее поведение `_refresh_day_rows` (инлайн-поля видимы у отмеченных
    в «Индивидуально»).
- `gui/main_window.py`:
  - Владеет `DayStatesWorker` (как `SmokeWorker`): ссылка `self._day_states_worker`.
  - Старт по `control.day_states_requested` (guard: не при активной операции/дубле).
  - `states_ready → control.apply_day_states`; `states_failed → control.apply_day_states_error`.

## Известные ограничения (указать в отчёте/доках)

- «Ремонт» закрытия дел (`complete_day_activities` в `run_fill`) идёт по уже заполненным дням
  окна независимо от галок — учёт не создаёт/не перезаписывает, только закрывает висящее дело;

  окно ≤5 дней — на практике совпадает.
