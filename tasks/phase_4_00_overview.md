# Фаза 4 — Создание учёта рабочего времени (FR-2.1.6–2.1.10.4)

**Статус:** TODO
**Зависит от:** Фаза 3
**Связано:** PRD §2.6, FR-2; PLAN «Этап 4».

## Цель фазы
Для отобранных дней создать дочерний элемент «Учёт рабочего времени» (1218) с привязкой к дню,
договором «Общие задачи подразделения», описанием и часами (дефолт + подтверждение).

## Результат фазы (DoD)
На одном дне (после явного разрешения на запись) создаётся учёт; привязка подтверждается
перечитыванием (`ufCrm46_1742997115` дня содержит новый ID). `--dry-run` показывает payload без записи.

## Подзадачи
1. `phase_4_01_done_button_investigation.md` — выяснить механизм «Выполнено» (read-only).
2. `phase_4_02_values_prompt.md` — дефолт + подтверждение значений.
3. `phase_4_03_create_log.md` — `crm.item.add` 1218 (plan→execute).
4. `phase_4_04_verify.md` — верификация результата.

## Важно
- По умолчанию режим «только чтение»: боевые записи в прод — лишь по явному разрешению заказчика.
- Строго соблюдать 4-дневное окно (проверка повторно перед записью).
