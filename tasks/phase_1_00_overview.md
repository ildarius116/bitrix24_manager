## Цель фазы
Запускаемый CLI-каркас проекта с рабочим доступом к порталу по REST (вебхук), на котором
будут строиться парсинг (Фаза 2) и редактирование (Фазы 3–4).

## Результат фазы (DoD)
- `python main.py --help` показывает подкоманды `export`, `fill`.
- Конфигурация читается из `.env` (секрет) + `config.yaml` (метамодель/дефолты).
- Есть обёртка `src/b24.py` поверх `.claude/skills/bitrix24-agent/scripts/bitrix24_client.py`.
- `user.current` отрабатывает (smoke), секрет маскируется в логах.

## Подзадачи
1. `phase_1_01_skeleton.md` — структура каталогов, загрузка конфигурации, CLI, логирование.
2. `phase_1_02_b24_wrapper.md` — тонкая обёртка над клиентом скилла.
3. `phase_1_03_smoke_access.md` — smoke-проверка доступа и понятные ошибки.

## Примечания
- Код пишется только по отдельному указанию (правило проекта).
- Никакого браузера: транспорт — только REST.
