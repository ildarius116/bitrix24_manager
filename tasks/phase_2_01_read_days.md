# phase_2_01 — Чтение дней «Рабочий день» (1208) за период

**Статус:** TODO
**Фаза:** 2 (Парсинг)
**Зависит от:** phase_1_02
**FR:** 1.1.1–1.1.3

## Цель
Считать все записи `entityTypeId=1208` за `[date_from; date_to]` с нужными полями.

## Объём работ
- `src/workday.py`: модель `WorkdayDay` (id, date, title, employee, works_ids, raw).
- Запрос `crm.item.list`:
  - `filter`: `{ ">=ufCrm46_1742342657": date_from, "<=ufCrm46_1742342657": date_to }`;
  - `order`: `{ "id": "desc" }`;
  - `select`: id, title, `ufCrm46_1742342657` (дата), `ufCrm46_1742997115` (работы), `ufCrm46_1742341577` (сотрудник).
  - полная пагинация (`item_list_all`).
- Дата дня — из `ufCrm46_1742342657`; запасной парсер — из `title` (`extract_date`).

## Критерии приёмки (DoD)
- Возвращается корректное число дней за период; `works_ids` заполнен из `ufCrm46_1742997115`.
- На периоде с >50 днями пагинация не теряет записи.
